# Rencana Perbaikan Indexing Google Search Console

## Tujuan
Meningkatkan peluang halaman `petshop.html`, `konveksi.html`, `pos.html`, `pesantren.html`, dan `cv.html` agar berstatus **Indexed** (bukan *Crawled - currently not indexed*) dengan memperkuat kualitas konten, sinyal teknis SEO, dan keterkaitan internal.

## Analisa Awal (berdasarkan file saat ini)
- Kelima halaman sudah memiliki fondasi SEO dasar: `title`, `meta description`, canonical, OG/Twitter, JSON-LD, dan konten fitur.
- Penyebab umum status *Crawled - currently not indexed* yang paling relevan di kasus ini:
  1. **Kualitas/keunikan konten dinilai belum cukup kuat** (halaman mirip pola template, struktur sangat seragam).
  2. **Intent pencarian & kedalaman konten** belum cukup menjawab kebutuhan pengguna (konten cenderung ringkas promosi).
  3. **Sinyal internal linking** antar halaman detail kemungkinan masih kurang kuat.
  4. **Kepercayaan halaman (E-E-A-T)** bisa ditingkatkan (bukti pengalaman, studi kasus, FAQ kontekstual, detail implementasi).
- Jadi, bukan semata “kurang panjang”, tetapi lebih ke **nilai unik + kelengkapan konteks + diferensiasi antar halaman**.

## Strategi Perbaikan

### 1) Perkuat konten unik tiap halaman project
Untuk setiap halaman (`petshop`, `pos`, `konveksi`, `pesantren`) tambahkan section artikel yang benar-benar berbeda dan spesifik:
- Latar belakang masalah bisnis per domain.
- Solusi sistem per modul + alur kerja nyata.
- Dampak bisnis yang terukur (contoh KPI: waktu proses, akurasi stok, error input).
- Use case harian (contoh skenario operasional).
- Siapa pengguna sistem (owner, kasir, admin, bendahara, operator).

Target: tiap halaman punya narasi unik, bukan hanya daftar fitur.

### 2) Tambahkan FAQ spesifik per halaman + schema FAQPage
- Buat 4-6 FAQ yang relevan per domain (berbeda antar halaman).
- Sinkronkan dengan JSON-LD `FAQPage` pada masing-masing file.
- Hindari FAQ generik yang sama persis antar halaman.

Target: memperkaya konteks semantik dan intent long-tail.

### 3) Tambahkan “proof” / kredibilitas
- Section ringkas: pengalaman 8+ tahun, 50+ project, pendekatan implementasi, support paska go-live.
- Tambahkan blok “proses kerja” (discovery → development → training → support).

Target: meningkatkan sinyal trust dan kualitas konten.

### 4) Perbaiki internal linking kontekstual
- Di setiap halaman detail, tambahkan:
  - Link ke 3 halaman solusi lain (related solutions).
  - Link ke `cv.html` sebagai profil/otoritas pembuat.
  - Link balik ke `index.html` (sudah ada, dipertahankan).
- Anchor text dibuat deskriptif (bukan hanya “klik di sini”).

Target: membantu Google memahami struktur topik situs.

### 5) Tambahkan metadata artikel yang lebih kuat
Pada setiap halaman project:
- Perbarui/cek JSON-LD `Article`:
  - `mainEntityOfPage`
  - `dateModified`
  - `inLanguage`
  - `keywords`
- Tambahkan meta `article:published_time` dan `article:modified_time` (Open Graph extension).

Target: sinyal freshness dan struktur data lebih lengkap.

### 6) Optimasi `cv.html` agar lebih indeksable sebagai halaman informasi
- Tambahkan section “Ringkasan Profesional” dengan konten naratif 2-4 paragraf.
- Tambahkan section “Layanan Utama” dan “Bidang Industri” (sinkron dengan halaman project).
- Tambahkan FAQ singkat CV (opsional) + schema bila dipakai.

Target: `cv.html` tidak hanya dianggap halaman utilitas/download.

### 7) QA teknis sebelum submit ulang
- Validasi tidak ada canonical ganda/salah.
- Pastikan tiap halaman return 200 OK (nanti cek saat eksekusi).
- Cek konsistensi title & meta description agar unik antar halaman.
- Cek schema valid (nanti pakai validator saat eksekusi manual).

## Rencana Implementasi Per File
- `petshop.html`: tambah 3-4 section artikel domain petshop + FAQ + FAQ schema + related links.
- `pos.html`: tambah 3-4 section artikel domain retail/POS + FAQ + FAQ schema + related links.
- `konveksi.html`: tambah 3-4 section artikel domain produksi konveksi + FAQ + FAQ schema + related links.
- `pesantren.html`: tambah 3-4 section artikel domain keuangan pesantren + FAQ + FAQ schema + related links.
- `cv.html` (opsional kuat): tambah narasi profesional + layanan + industry coverage + internal links ke seluruh halaman project.

## Setelah Deploy (langkah operasional di GSC)
1. Gunakan **URL Inspection** per halaman → Request Indexing.
2. Submit ulang `sitemap.xml`.
3. Tunggu 3-14 hari; pantau status coverage per URL.
4. Jika masih *Crawled - currently not indexed*, evaluasi lagi halaman dengan impresi rendah dan tambah kedalaman konten/intent.

## Catatan Penting
- Status ditolak berulang di GSC tidak selalu error teknis; sering karena sistem Google belum melihat nilai pembeda yang cukup.
- Fokus utama perbaikan: **keunikan konten + relevansi intent + internal link + trust signals**.

## Kriteria Sukses
- Tiap halaman memiliki konten unik yang substansial dan tidak template-heavy.
- Schema valid dan sesuai isi halaman.
- Internal linking antar topik terbentuk jelas.
- Dalam beberapa siklus crawl, URL beralih ke status Indexed atau setidaknya menunjukkan progres impresi di Search Console.
