# Rencana Perbaikan Indexing Google Search Console

## Tujuan
Meningkatkan peluang halaman `petshop.html`, `konveksi.html`, `pos.html`, `pesantren.html`, dan `cv.html` agar berstatus **Indexed** (bukan *Crawled - currently not indexed*) dengan memperkuat kualitas konten, sinyal teknis SEO, dan keterkaitan internal.

## Status Saat Ini (sudah selesai)
Perbaikan utama untuk halaman detail dan CV sudah diterapkan sebelumnya:
- Penambahan konten artikel unik per halaman project.
- Penambahan FAQ visual + JSON-LD FAQPage per halaman project.
- Penguatan metadata `Article` (`mainEntityOfPage`, `dateModified`, `inLanguage`, `keywords`).
- Penambahan `article:published_time` dan `article:modified_time`.
- Penambahan internal link antar halaman project + ke `cv.html`.
- Pembaruan `sitemap.xml` pada field `lastmod`.

## Fokus Lanjutan (sesuai permintaan terbaru)
Fokus implementasi berikut hanya pada 3 hal:
1. **Kurangi kode tidak terpakai** di `index.html`.
2. **Tambahkan internal link kontekstual** di `index.html`.
3. **Perkuat metadata freshness** di `index.html`.

## Rencana Implementasi Detail untuk `index.html`

### 1) Kurangi kode tidak terpakai
- Hapus fungsi JavaScript yang tidak dipakai:
  - `toggleDescription(...)`
  - `toggleCards(...)`
  Kedua fungsi ini tidak punya elemen target aktif pada halaman.
- Hapus injeksi CSS `.hidden-card` yang tidak terpakai.
- Rapikan blok komentar besar yang sudah tidak digunakan (opsional aman):
  - blok nav yang dicomment,
  - blok FAQ visual lama yang dicomment,
  - blok footer yang dicomment.

Target:
- HTML/JS lebih bersih, lebih mudah dipelihara, dan mengurangi sinyal kualitas rendah akibat dead code.

### 2) Tambahkan internal link kontekstual
- Tambahkan section baru setelah daftar project / sebelum CTA, misalnya: **“Jelajahi Solusi Berdasarkan Kebutuhan”**.
- Isi section berupa link teks deskriptif ke:
  - `petshop.html` (anchor: “Sistem Petshop untuk operasional penjualan, grooming, dan klinik”)
  - `pos.html` (anchor: “Sistem POS untuk kasir, stok, dan laporan penjualan”)
  - `konveksi.html` (anchor: “Sistem Produksi Konveksi dari order hingga finishing”)
  - `pesantren.html` (anchor: “Sistem Keuangan Pesantren untuk SPP, donasi, dan akuntansi”)
  - `cv.html` (anchor: “Profil dan pengalaman lengkap developer”)
- Pertahankan link “Lihat Detail” yang sudah ada, tetapi tambah konteks di section baru supaya semantic linking lebih kuat.

Target:
- Struktur topik situs lebih jelas untuk crawler.
- Distribusi internal link lebih natural dan kaya intent.

### 3) Perkuat metadata freshness
- Tambahkan pada `<head>`:
  - `<meta property="og:updated_time" content="2026-05-15">`
- Tambahkan JSON-LD `WebPage` atau `WebSite` (lebih direkomendasikan `WebPage` untuk homepage), mencakup:
  - `@type`: `WebPage`
  - `url`: homepage
  - `name`
  - `inLanguage`
  - `dateModified`
  - `description`
- Pertahankan schema existing (`Person`, `FAQPage`, `ProfessionalService`) agar tidak bentrok.
- Sinkronkan `sitemap.xml` untuk homepage (`/`) `lastmod` menjadi tanggal update terkini bila belum sama.

Target:
- Sinyal “konten diperbarui” lebih jelas untuk mesin pencari.
- Entitas homepage lebih lengkap secara schema.

## QA Setelah Implementasi
- Cek tidak ada error console JS akibat penghapusan fungsi lama.
- Cek seluruh internal link baru mengarah ke file yang benar.
- Cek validitas schema via Rich Results Test / Schema Validator.
- Cek `view-source` memastikan `og:updated_time` dan schema baru terbaca.

## Langkah Operasional GSC Setelah Deploy
1. Submit ulang `sitemap.xml`.
2. URL Inspection untuk `https://fauzicihuyy.web.id/` lalu Request Indexing.
3. Tunggu recrawl dan pantau status Coverage + Performance selama 3–14 hari.

## Kriteria Sukses
- `index.html` bebas dead code utama yang tidak terpakai.
- Internal link kontekstual bertambah dengan anchor deskriptif.
- Metadata freshness homepage terbaca jelas (`og:updated_time`, `dateModified` schema, `lastmod` sitemap).
- Dalam siklus crawl berikutnya, sinyal kualitas teknis homepage meningkat di GSC.
